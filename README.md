## Descripción del proyecto

Este proyecto fue desarrollado con el propósito de comparar dos enfoques de programación en Python: la Programación Tradicional y la Programación Orientada a Objetos (POO).

Para ello se implementó un sistema básico de gestión de mascotas que permite registrar y mostrar información de una mascota utilizando ambos paradigmas.

Los datos manejados son:

* Nombre
* Especie
* Edad

---

## Programa 1: Programación Tradicional

El programa tradicional fue desarrollado utilizando funciones, variables y entrada de datos por teclado.

Características principales:

* Utiliza funciones para organizar las tareas.
* Solicita información de la mascota mediante teclado.
* Almacena los datos en una estructura tipo diccionario.
* Muestra la información registrada de manera organizada.
* No utiliza clases ni objetos.

Archivo:

* `programacion_tradicional/tradicional.py`

---

## Programa 2: Programación Orientada a Objetos (POO)

El programa orientado a objetos fue desarrollado utilizando una clase llamada `Mascota`.

Características principales:

* Implementa atributos: nombre, especie y edad.
* Utiliza métodos para mostrar información y representar comportamientos.
* Crea varios objetos de la clase Mascota.
* Organiza los datos y comportamientos dentro de una misma estructura.

Archivos:

* `programacion_poo/mascota.py`
* `programacion_poo/main.py`

---

## Reflexión

Durante el desarrollo de esta actividad pude observar que la Programación Tradicional se centra en las funciones y procedimientos que realizan acciones sobre los datos. Este enfoque resulta adecuado para programas pequeños y sencillos debido a su simplicidad.

Por otro lado, la Programación Orientada a Objetos organiza el software mediante clases y objetos, integrando datos y comportamientos dentro de una misma estructura. Esto facilita la organización del código, su reutilización y su mantenimiento cuando los sistemas crecen en complejidad.

En conclusión, ambos paradigmas permiten resolver problemas computacionales; sin embargo, la Programación Orientada a Objetos ofrece mayores ventajas para proyectos de mediana y gran escala debido a su capacidad de organización, reutilización y escalabilidad.
